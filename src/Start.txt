-. Crear el siguiente alias de base de datos desde Sql Server Configuration

Alias Name: GuideDbServer
Pipe Name: \\.\pipe\MSSQL$sqlexpress\sql\query 
Protocol: Named Pipes
Server: .

-. Verificar la configuración de las siguientes Propiedades del proyecto Framework

Windows Authentication: Disabled
Anonymous Authentication: Enabled

-. En VS2012 abrir View -> Other Windows -> Manage Nuget Package, seleccionar en el combo el proyecto Model.

-. Ejecutar en la línea de comando update-database -force. Esto actualizará el modelo con la base de datos.

-. Correr el script de base de datos que cargará los maestros desde le proyecto Model.SQL -> script PostDeployment.Masters.

-. Ejecutar Redis en 
		\tools\bin\redis-2.4.5\64bit\redis.exe

-. Ejecutar Solr en
	
	\tools\bin\solr-1.4.0\runsolr.bat

	Para ver la consola:

	http://localhost:8983/Solr

-. Configuración de SQLExpress para que el DataImportHandler consuma los datos de la base (opcional)

		. Verificar que el firewall no bloquee al servidor de SQL (UDP traffic to port 1434/1433)

		. Crear la carpeta lib dentro del core utilizado y dentro copiar los archivos 

			solr-dataimporthandler-4.2.1.jar (ubicado en la carpeta Dist de Solr)
			solr-dataimporthandler-extras-4.2.1.jar (ubicado en la carpeta Dist de Solr)
			sqljdbc4.jar (bajarlo del sitio de Microsoft)

		. Verificar que esté corriendo el servicio SQL Server Browser Service

		. En Sql Server Configuration Manager habilitar la conección TCP

-. Correr la app.
